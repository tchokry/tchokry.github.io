  • Tools: CodeLab, Microsoft Visual Studio, Github

Project Overview

The goal of this project is to create an interactive social media program to connect villains in our data set. Similar to any other social media system, each villain will be able to interact with each other by using features such as adding and removing friends, messages, friend recommendation, and search for friends. They will also have their own login information along with some personal details that will be displayed on their profile.

Topic Summary

A villain is an "evil" character in a story, whether it's in movies, series shows, or comic books. The villain usually is the antagonist - the character who tends to harm other characters. We chose this topic because, without the villains, there is rarely a plot of the story. Villains are often better-written characters, with behavioral nuances and eccentricities, whereas the "hero" characters are usually one dimensional, all good, all white. That said, villains provide another dark side of the story to balance the power of the hero.

Data Set

Each of our data set contain personal information of the villains such as their first and last name, list of friends, personal interest, and login details. These personal interests include skydiving, hunting, boxing, wrestling, gambling, racing, and parkour. All of the information presented in our data set was obtained from sources such as Villains Wikia and Rolling Stones to ensure credibility.

User Testing

To gain user feedbacks, we execute user testing in which we present three scenario that user can utilize to explore our program.





User Feedback

After conducting the user testing, we then gain some insights to what we can further improve in the final iteration of our program. Below are some feedbacks and problem that we encounter during testing:

  • User prefer the search field to be string in which user can type in the name or interest instead of integers as its mirrors the traditional social media features the most

  • Some users encounter difficulties on following the scenarios we have written because it's too complicated. Thus, we have simplified those scenario with bullet points so users can read and follow it with more ease.

  • One of the users suggested for us to add another features such as "Rating", in which each villains could rate each other. The reason behind this is so we can add more characteristic to our simple program
